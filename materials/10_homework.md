# Домашнее задание

Расчетное время: 30-45 минут.

## Цель

Закрепить навык использования ИИ как помощника QA: сначала самостоятельно подумать над проверками, затем получить черновик от ИИ, найти ошибки и оформить выводы.

Главное правило:

```text
ИИ делает черновик, QA проверяет.
```

## Что нужно сделать

1. Выбрать одно мини-требование из списка ниже или придумать свое учебное требование.
2. Самостоятельно составить 6-10 проверок.
3. Попросить ИИ улучшить или дополнить набор проверок.
4. Найти в ответе ИИ:
   - полезные идеи;
   - пропуски;
   - лишние проверки;
   - неподтвержденные допущения;
   - неясные expected result.
5. Оформить итоговый набор проверок.
6. Написать короткий вывод: что ИИ сделал хорошо, где ошибся, что проверял QA.

## Мини-требования на выбор

### Вариант 1. Форма входа

```text
Пользователь может войти по email и паролю.
Email обязателен и должен быть валидным.
Пароль обязателен.
При успешном входе пользователь попадает на главную страницу.
Если email или пароль неверны, показывается сообщение «Неверный email или пароль».
```

### Вариант 2. Поиск товара

```text
Пользователь может искать товар по названию.
Если товары найдены, отображается список результатов.
Если товары не найдены, отображается сообщение «Ничего не найдено».
Пустой поисковый запрос не отправляется.
```

### Вариант 3. Фильтр заказов

```text
Пользователь может отфильтровать заказы по статусу: Новый, В работе, Завершен, Отменен.
Можно выбрать только один статус одновременно.
После выбора статуса список заказов обновляется.
Если заказов с выбранным статусом нет, показывается сообщение «Заказы не найдены».
```

## Промпт для ИИ

```text
Ты помогаешь начинающему QA улучшить набор проверок.

Требование:
[вставь выбранное мини-требование]

Мой черновик проверок:
[вставь свои 6-10 проверок]

Задача:
1. Проверь мой набор.
2. Найди пропущенные позитивные, негативные и граничные сценарии.
3. Укажи лишние проверки, если они не следуют из требования.
4. Не выдумывай требований.
5. Если данных не хватает, напиши «Недостаточно данных».
6. Если делаешь предположение, пометь его как «Допущение».

Формат:
- Что хорошо в моем наборе
- Что пропущено
- Что лишнее или неподтвержденное
- Итоговый набор проверок
- Что QA должен проверить вручную
```

## Критерии оценки

| Критерий | Что проверяется |
|---|---|
| Самостоятельный старт | Студент сначала составил свои проверки, а не начал с ИИ |
| Покрытие | Есть позитивные и негативные проверки |
| Проверка ИИ | Студент нашел полезные идеи, пропуски и сомнительные места |
| Работа с нехваткой данных | Неизвестные факты помечены как «Недостаточно данных» |
| Отсутствие выдуманных требований | Итоговый набор не содержит функций, которых нет в требовании |
| Безопасность | Нет реальных персональных данных, паролей, токенов, приватных материалов |
| Вывод | Студент объяснил, где ИИ помог и где требовалась проверка QA |

## Вариант без доступа к ИИ

Если нет доступа к ИИ-сервису, студент анализирует готовый ответ ниже.

### Требование

```text
Пользователь может искать товар по названию.
Если товары найдены, отображается список результатов.
Если товары не найдены, отображается сообщение «Ничего не найдено».
Пустой поисковый запрос не отправляется.
```

### Готовый ответ ИИ для ревью

```text
1. Проверить поиск существующего товара.
2. Проверить поиск несуществующего товара.
3. Проверить пустой запрос.
4. Проверить поиск по категории.
5. Проверить сортировку результатов по цене.
6. Проверить добавление найденного товара в корзину.
7. Проверить поиск по части названия.
8. Проверить сообщение «Ничего не найдено».
```

### Задача студента

1. Найти проверки, которые следуют из требования.
2. Найти проверки, которые не подтверждены требованием.
3. Найти, каких данных не хватает.
4. Составить итоговый исправленный набор.

### Подсказка для самопроверки

В готовом ответе есть полезные проверки, но есть и неподтвержденные функции: категории, сортировка, корзина. Проверка поиска по части названия может быть полезной идеей, но в требовании это не описано, значит нужно пометить как «Недостаточно данных» или вопрос к аналитику.

## Ограничения

- Не использовать реальные данные пользователей.
- Не отправлять в ИИ закрытые рабочие требования.
- Не копировать ответ ИИ без собственного ревью.
- Не считать платный инструмент гарантией правильного ответа.
