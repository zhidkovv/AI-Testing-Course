# Сравнение ИИ-инструментов для QA

Дата проверки: 28.06.2026.

Важно: цены, лимиты, модели и доступность функций быстро меняются. Эта таблица не является рекомендацией купить конкретный сервис. Ее задача - показать студентам, по каким критериям сравнивать ИИ-инструменты для QA-задач.

Если цена или лимит не найдены в официальном источнике на дату проверки, это явно указано в таблице. Не нужно заменять такие пробелы догадками.

## Как читать таблицу

- «Бесплатный доступ» означает наличие бесплатного режима или пробного доступа, подтвержденного официальной страницей.
- «Платные уровни» указаны только по официальному источнику.
- Для региональных цен указана валюта страницы, которая открылась при проверке. Не конвертировать цены без отдельной проверки.
- Для QA важнее не «самая сильная модель», а проверяемый результат, работа с исходными данными, безопасность и удобство ревью.

## Выводы для занятия

1. Бесплатного доступа часто достаточно для учебных черновиков: тест-кейсы, чек-листы, баг-репорты, вопросы к требованиям.
2. Платные уровни обычно дают больше лимитов, контекста, файлов, поиска, агентных функций, интеграций или приоритета.
3. Цена и лимит не равны качеству QA-результата.
4. Любой ответ ИИ нужно проверять по требованиям.
5. Если официальный источник не подтверждает функцию, ее нельзя подавать студентам как факт.
6. Для реального проекта важны не только возможности модели, но и правила обработки данных.

## Что показать на занятии за 12 минут

Не нужно подробно разбирать все строки таблицы. Рекомендуемый сценарий:

1. Показать 5 критериев выбора: лимиты, файлы, web search, приватность, агентные функции.
2. Сравнить один бесплатный и один платный режим без рекламы конкретного сервиса.
3. Показать, что платный доступ не отменяет чек-лист проверки ответа.
4. Попросить студентов выбрать, какие критерии важны для ручного QA.

## Что не делать

- Не заставлять студентов покупать подписку.
- Не утверждать, что платный инструмент «лучше тестирует».
- Не переписывать маркетинговые заявления как доказанные факты.
- Не использовать неподтвержденные цены из блогов и обзоров.
- Не сравнивать модели только по бенчмаркам.
